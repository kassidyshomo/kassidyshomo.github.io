Professional Self-Assessment

This ePortfolio is the final capstone project for my Bachelor of Science in Computer Science at SNHU, with a concentration in Software Engineering. In this course, I selected three artifacts from prior coursework and enhanced them to demonstrate growth across software engineering and design, algorithms and data structures, and database systems. Each artifact was improved with a focus on structure, efficiency, and security, with an emphasis on applying real-world development practices rather than only basic functionality. This self-assessment explains how those enhancements, along with my program experience, demonstrate my ability to meet the core course outcomes and apply them in practical technical work.

The core course outcomes for this capstone are:

  • Employ strategies for building collaborative environments that support organizational decision-making in computer science
  • Design, develop, and deliver professional-quality technical communication for both technical and non-technical audiences
  • Design and evaluate computing solutions using algorithmic principles while managing design trade-offs
  • Apply computing tools, techniques, and practices to build systems that deliver value in real environments
  • Develop a security mindset that identifies risks, reduces vulnerabilities, and protects systems and data

Across the program and in my current role as a Business Support Analyst, I have developed a stronger ability to work with technical information in a way that supports communication and decision-making between different stakeholders. This directly reflects the program outcome focused on building collaborative environments that support organizational decision-making in computer science. In my role, I work with technical documentation, system requirements, and backend processes across multiple platforms, often translating technical details into clear operational understanding. This experience has also strengthened my ability to communicate complex systems in a structured and accessible way, which aligns with the outcome focused on professional-quality technical communication for both technical and non-technical audiences. That same skill carries into my academic work when documenting design decisions and explaining system behavior in my artifacts and narratives.

In my buffer overflow artifact, I focused on improving memory safety and input handling. The original program allowed user input to overflow a fixed buffer, which could corrupt memory. In the enhanced version, I added input validation before any data is written to memory and separated validation logic from the main program flow. This improves structure, readability, and safety. This work aligns with outcomes related to algorithmic problem solving, software design, and security because it shows how unsafe input can be prevented through design instead of handled after failure. It also reflects a security mindset by focusing on preventing memory-based vulnerabilities before they occur.

In my numeric overflow and underflow artifact, I improved the efficiency of repeated loop-based calculations. The original version used iteration to simulate arithmetic limits. In the enhanced version, I replaced repeated loops with direct computation and added pre-check validation to prevent unsafe arithmetic operations. This reduced time complexity from O(n) to O(1) while maintaining correctness. This aligns with outcomes focused on algorithm design, trade-off analysis, and applying efficient computing practices. It also shows an understanding of how design choices affect both performance and correctness in real systems.

In my database artifact, I improved how the program handles SQL queries and user input. The original version demonstrated how unsafe string concatenation could lead to SQL injection. In the enhanced version, I replaced pattern-based detection with parameterized queries using prepared statements so user input is always treated as data. This change improves both security and reliability. This aligns directly with the security outcome because it prevents exploitation by removing unsafe query construction. It also supports software engineering practices by improving structure and making database interaction more consistent and maintainable.

Security is a consistent theme across all three artifacts. In each case, I focused on identifying a weakness in the original design and improving it through safer structure rather than reactive checks. Whether it was memory safety, arithmetic limits, or database queries, the goal was always to reduce risk through better design. This reflects a security mindset that looks for how systems can fail and addresses those issues at the design level instead of after errors occur.

Overall, this portfolio shows my growth throughout the Computer Science program. I have moved from focusing mainly on making programs work to focusing on how they are designed, how they handle edge cases, and how they can be made more secure and efficient. These skills will continue to guide my work in software development and technical problem solving, especially in environments where reliability, structure, and clear communication are important.

Artifacts

The artifacts below were originally developed in CS 405: Secure Coding and later enhanced in CS 499. Each one focuses on a different core area of computer science and demonstrates applied improvement in security, structure, or efficiency.

Software Engineering and Design

The original Buffer Overflow program demonstrates a critical vulnerability where user input larger than a fixed buffer size can overwrite adjacent memory. This type of issue is a well-known security risk in low-level programming and can lead to unpredictable behavior or system compromise.

The enhanced version resolves this issue by introducing structured input validation before memory allocation. Instead of allowing unsafe data into the buffer and reacting to failure afterward, the program now prevents invalid input from ever being written into memory. This improves both security and reliability while also making the program structure easier to maintain and understand.

Algorithms and Data Structures

The original Numeric Overflow program uses iterative loops to simulate arithmetic operations that can exceed the limits of a given data type. While this effectively demonstrates how overflow and underflow occur, it is inefficient because it relies on repeated computation rather than direct calculation.

The enhanced version improves both performance and structure by replacing iterative logic with direct mathematical computation. It also introduces pre-check validation to ensure that values remain within safe numeric boundaries before execution. This reduces time complexity while preserving correctness and improving overall efficiency.

Databases

The SQL Injection artifact demonstrates how unsafe string concatenation can allow malicious input to alter the structure of a database query. This is a critical vulnerability in database-driven applications because it can expose or modify sensitive data.

The enhanced version eliminates this risk by using parameterized queries, ensuring that user input is always treated as data rather than executable SQL. This change significantly improves security by preventing query manipulation and aligns the program with standard database security practices used in industry systems.
