Professional Self-Assessment

This ePortfolio is the final capstone project for my Bachelor of Science in Computer Science at SNHU, with a concentration in Software Engineering. In this course, I selected three artifacts from prior coursework and enhanced them to demonstrate growth across software engineering and design, algorithms and data structures, and database systems. Each artifact was improved with a focus on structure, efficiency, and security, with an emphasis on applying real-world development practices rather than only basic functionality. This self-assessment explains how those enhancements, along with my program experience, demonstrate my ability to meet the core course outcomes and apply them in practical technical work.

Course Outcomes

  1. Employ strategies for building collaborative environments that enable diverse audiences to support organizational decision-making in the field of computer science.
  2. Design, develop, and deliver professional-quality oral, written, and visual communications that are coherent, technically sound, and appropriately adapted to specific audiences and contexts.
  3. Design and evaluate computing solutions that solve a given problem using algorithmic principles and computer science practices and standards appropriate to its solution while managing the trade-offs involved in design choices.
  4. Demonstrate an ability to use well-founded and innovative techniques, skills, and tools in computing practices for the purpose of implementing computer solutions that deliver value and accomplish industry-specific goals.

One of the biggest areas of growth for me has been communication and technical understanding in a real systems environment, aligning with Outcomes 1 and 2. In my role as a Business Support Analyst, I work with contracts, technical documentation, and system requirements, often translating and interpreting technical language related to hardware, software, networking, and application-level systems. I also handle backend project setup across different systems, which requires understanding how components connect and function together. This experience has strengthened my ability to communicate technical concepts clearly and support organizational decision-making using accurate system information. It also aligns with Outcome 2 through my ability to communicate complex systems in a structured and audience-appropriate way, both professionally and academically.

My development in software engineering and design is demonstrated in my buffer overflow artifact, which aligns with Outcomes 3 and 5. The original program allowed user input to overflow a fixed buffer, creating a memory safety vulnerability. In the enhanced version, I implemented input validation before data is written to memory and separated validation logic from the main program flow. This improves structure, readability, and safety by ensuring invalid input is handled before it can impact program execution. This reflects Outcome 3 through improved design decisions and trade-off awareness, and Outcome 5 through preventing memory-based vulnerabilities through secure design rather than reactive error handling.

In my numeric overflow and underflow artifact, I improved algorithmic efficiency and logic design, aligning with Outcomes 3 and 4. The original version relied on iterative loops to simulate arithmetic limits, which introduced unnecessary computational overhead. In the enhanced version, I replaced this loop-based approach with direct computation and added pre-check validation to ensure safe execution. This reduced time complexity from O(n) to O(1) while maintaining correctness. This reflects Outcome 3 through algorithmic improvement and trade-off evaluation, and Outcome 4 through the effective use of computing techniques that improve performance and maintain reliability.

My database-related artifact demonstrates Outcome 5 through improved security and data handling practices. The original version used unsafe string handling patterns that could expose the system to SQL injection vulnerabilities. In the enhanced version, I replaced these patterns with parameterized queries using prepared statements, ensuring that user input is treated strictly as data rather than executable logic. This change improves system security and aligns with industry standards for secure database design. It also reflects Outcome 4 through the use of established development tools and practices that improve system reliability and maintainability.

Security is a consistent theme across all three artifacts and directly reflects Outcome 5. In each case, I focused on identifying vulnerabilities in the original design and addressing them through structural improvements rather than reactive fixes. Whether dealing with memory safety, arithmetic overflow, or database security, the goal was always to reduce risk through better design decisions and safer system architecture. This approach demonstrates a consistent security mindset focused on anticipating and preventing issues at the design level rather than responding after failure occurs.

Overall, this portfolio demonstrates my growth throughout the Computer Science program and how I apply these skills in professional practice. I have progressed from focusing primarily on functional programming to focusing on how systems are structured, how they handle edge cases, and how they can be made more secure and efficient. This work reflects my ability to meet all five program outcomes through technical implementation, algorithmic reasoning, secure design practices, communication, and professional experience. These skills will continue to guide my work in software development and technical problem solving, especially in environments where reliability, clarity, and security are essential.

Artifacts

The artifacts below were originally developed in CS 405: Secure Coding and later enhanced in CS 499. Each one focuses on a different core area of computer science and demonstrates applied improvement in security, structure, or efficiency.

Software Engineering and Design

The original Buffer Overflow program demonstrates a critical vulnerability where user input larger than a fixed buffer size can overwrite adjacent memory. This type of issue is a well-known security risk in low-level programming and can lead to unpredictable behavior or system compromise.

The enhanced version resolves this issue by introducing structured input validation before memory allocation. Instead of allowing unsafe data into the buffer and reacting to failure afterward, the program now prevents invalid input from ever being written into memory. This improves both security and reliability while also making the program structure easier to maintain and understand.

Algorithms and Data Structures

The original Numeric Overflow program uses iterative loops to simulate arithmetic operations that can exceed the limits of a given data type. While this effectively demonstrates how overflow and underflow occur, it is inefficient because it relies on repeated computation rather than direct calculation.

The enhanced version improves both performance and structure by replacing iterative logic with direct mathematical computation. It also introduces pre-check validation to ensure that values remain within safe numeric boundaries before execution. This reduces time complexity while preserving correctness and improving overall efficiency.

Databases

The SQL Injection artifact demonstrates how unsafe string concatenation can allow malicious input to alter the structure of a database query. This is a critical vulnerability in database-driven applications because it can expose or modify sensitive data.

The enhanced version eliminates this risk by using parameterized queries, ensuring that user input is always treated as data rather than executable SQL. This change significantly improves security by preventing query manipulation and aligns the program with standard database security practices used in industry systems.
